Authentication

POST /api/auth/register

Register a new user account

Request Body:

{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john@example.com",
  "password": "securePassword123"
}

Response (200 OK):

{
  "record": {
    "id": "uuid",
    "firstName": "John",
    "lastName": "Doe",
    "email": "john@example.com",
    "verified": false
  },
  "msg": "User successfully created"
}
POST /api/auth/login

Login with email and password

Request Body:

{
  "email": "john@example.com",
  "password": "securePassword123"
}

Response (200 OK):

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "uuid",
    "email": "john@example.com",
    "fname": "John",
    "lname": "Doe",
    "role": "user",
    "currency": "USD",
    "subscribed": false
  }
}
POST /api/auth/adminlogin
Admin

Admin/SuperAdmin login endpoint

Request Body:

{
  "email": "admin@example.com",
  "password": "adminPassword123"
}

Note: This endpoint requires the user's role to be 'admin' or 'superadmin'

GET /api/auth/verify-email?token={token}

Verify user email address using verification token

POST /api/auth/forgot-password

Request password reset OTP

Request Body:

{
  "email": "john@example.com"
}

Response (200 OK):

{
  "msg": "Password reset OTP sent to your email",
  "resetToken": "reset-token-here"
}
POST /api/auth/verify-otp

Verify OTP for password reset

Headers:

x-reset-token: reset-token-from-forgot-password

Request Body:

{
  "otp": "1234"
}
POST /api/auth/reset-password

Reset password with new password

Headers:

x-reset-token: reset-token-from-forgot-password

Request Body:

{
  "newPassword": "newSecurePassword123"
}

Users

GET /api/users

Get all users with pagination and search

Query Parameters:

  • page - Page number (default: 1)
  • limit - Items per page (default: 10)
  • search - Search by firstName, lastName, or email

Response (200 OK):

{
  "users": [...],
  "totalCount": 100,
  "totalPages": 10,
  "currentPage": 1,
  "hasNextPage": true,
  "hasPrevPage": false
}
GET /api/users/:id

Get user details by ID including reviews

PUT /api/users/:id

Update user information

PUT /api/users/profile/:id

Enhanced profile update with validation

Request Body (all fields optional):

{
  "firstName": "John",
  "lastName": "Doe",
  "email": "newemail@example.com",
  "phone": "+1234567890",
  "bio": "User bio",
  "location": "New York",
  "currentPassword": "oldPassword",
  "newPassword": "newPassword"
}
PUT /api/users/:id/subscribe

Manually subscribe a user (admin action)

Admin Action
PUT /api/users/:id/unsubscribe

Manually unsubscribe a user (admin action)

Admin Action
DELETE /api/users/:id

Delete a user account

Admin Management

POST /api/admin/team-members
üîí Auth Required

Create a new admin or superadmin team member

Communities

POST /api/communities/create

Create a new community

GET /api/communities

Get all communities with pagination, filters, and subscription check

Query Parameters:

  • page - Page number (default: 1)
  • limit - Items per page (default: 10)
  • search - Search by name or description
  • communityType - Filter by community type
  • size - Filter by size
  • engagementLevel - Filter by engagement level
  • userId - Current user ID for subscription check

Subscription Check: Non-subscribed users see limited results

GET /api/communities/:id

Get community details by ID

PUT /api/communities/:id

Update community information

PUT /api/communities/restrict/:id

Toggle community restriction status

GET /api/communities/:userId/communities

Get all communities created by a specific user

DELETE /api/communities/:id

Delete a community
